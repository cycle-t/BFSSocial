BFS 原理讲解界面

原理总览

广度优先搜索会先看“最近的一圈”，再逐层向外扩散。

在六度空间这类无权图问题里，BFS 的关键不是“搜得多快”，而是“层级顺序永远正确”。 只要节点第一次被发现，它距离起点的层数就已经确定；只要目标第一次出队，最短路径也就确定了。

01 Queue

核心结构

使用队列维护“下一批要访问的节点”，保证先进入的节点先被处理，天然形成逐层扩散。

02 Layer

层级顺序

起点是第 0 层，所有直接邻居会成为第 1 层，再往外一圈一圈推进，顺序不会乱。

03 Seen

去重机制

节点第一次被发现时就加入 `seen`，后续再次遇到直接跳过，避免重复搜索和环路来回。

04 Parent

路径回溯

每个新节点都记录“我是从谁来的”，等命中目标后顺着父节点回溯，就能还原完整最短链路。

05 Target

命中判定

在无权图里，目标第一次出队时就已经锁定最短路径，不需要再继续向更外层盲目扩散。

06 Analogy

课堂比喻

可以把 BFS 讲成“朋友圈扩散模型”：先朋友，再朋友的朋友，最后才是更远的关系层。

07 Complexity

复杂度视角

每个节点和每条边通常只会被有效处理一次，所以 BFS 的时间复杂度常写作 `O(V + E)`。

08 Teaching

教学落点

学生真正要抓住的不是公式，而是“按层扩散、首次发现、父节点回溯”这三条核心逻辑。

讲解抓手

把抽象图论翻译成学生熟悉的社交网络，会比直接讲队列和邻接表更容易进入情境。

  1. 先固定起点，标记为第 0 层。
  2. 从队头取出节点，逐个检查邻居。
  3. 邻居第一次出现就入队，并记录父节点。
  4. 目标出队后停止，再沿父节点回溯最短路径。

一页讲清 BFS

为什么是队列： 队列能保证先发现的一层先被处理，不会跳层。

为什么不会重复搜： 每个节点第一次出现就加入 `seen`，后面再遇到直接跳过。

为什么适合六度空间： 社交关系通常可以建模为无权图，最短关系链就是最少边数。

为什么要记父节点： 找到目标后，靠父节点回溯就能还原完整路径。

第一性原理

层级比路径更重要。BFS 先保证“距离正确”，再谈“路径回溯”。

BFS vs DFS

DFS 更像深挖一条路，适合遍历与回溯；BFS 更像同步扩散，适合最短路与层次结构。

    核心伪代码

    同步高亮
    1. queue <- [start], seen <- {start}
    2. parent[start] <- null, dist[start] <- 0
    3. while queue is not empty
    4. v <- dequeue(queue)
    5. if v == target: break
    6. for each neighbor u of v
    7. if u not in seen: parent[u]=v; enqueue(u)
    8. reconstruct path(target, parent)
